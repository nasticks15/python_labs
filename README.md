## Теоретична частина

### 1. Рівні тестування
* **Unit (Модульне):** перевірка окремих функцій.
* **Integration (Інтеграційне):** перевірка взаємодії модулів.
* **System (Системне):** перевірка всієї програми в зборі.
* **Acceptance (Приймальне):** перевірка на відповідність бізнес-вимогам.

### 2. Покриття коду (Code Coverage)
Це метрика, що показує відсоток коду, який виконується під час тестів. Результати звіту допомагають зрозуміти, які частини програми залишилися неперевіреними.

### 3. Роль Git
Git дозволяє вести історію змін, працювати в командах через гілки та інтегрувати автоматичне тестування (CI/CD).
